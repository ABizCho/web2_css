CSS예시

웹브라우저는 처음 나왔을 때 HTML 언어만을 처리하기 위해 존재했다.
따라서 웹브라우저는 해당 페이지상 표현된 모든 언어를 기본적으로 HTML 이라고 생각한다

따라서 우리는 CSS를 사용하고자 할 때 HTML이 아니라 CSS의 문법에 따라 해석하라고 HTML 문법으로 웹브라우저에게 알려줘야한다.

한 웹페이지를 구성하는 HTML 코드 스크립트의 헤더부분에(head)style태그를 추가시키고,
이 태그 사이에 body에서 CSS로 표현될 코드를 일러줌으로써, 웹브라우저에게 CSS로 해석될 부분을 사전에 알리는 것이다.

위에서 말한 방법을 조금만 더 구체적으로 기술하자면,
style 태그 내에

" tag명 { 속성:속성값; } "

과 같이 일러줌으로써 선언한 태그명을 가진 모든것들의 특정 속성에 대해 프로그래머가 일러준 속성값을 부여함으로써 해당 페이지 내에서 일괄적으로 스타일을 적용시켜줄 수 있다.

해당 웹페이지에선 'CSS란' 페이지에서 font태그를 사용해서 디자인을 꾸며준 소스코드를 주석처리한 후, CSS를 활용해서 페이지를 꾸며본다.

이렇게 CSS라는 새로운 언어의 문법을 통해 디자인을 꾸며보았다
자 CSS와 HTML태그의 활용 두가지 방법을 두고 보았을 때 무엇이 더 배우기 어려울까? 당연히 새로 배워야 하는 CSS이다
웹브라우저 역시 폰트태그는 그저 태그만 추가하면 되지만, css는 새로운 언어를 해석하는 기능을 탑재해야하기 때문에 부담이다.

그러나 이러한 두가지 측면의 어려움에도 불구하고 CSS가 널리 사용되고 사용해야만 하는, 중복작업 제거라는 효율성과 의도를 전임자의 태그의도를 해치지 않는다는 효과성 이라는 폭발적인 시너지가 존재하기 때문이다.

이러한 관점에서, HTML에게 그 본질의 목적인 정보의 전달을 전념시키기 위해, CSS를 등장시키고 이에 부가적인 스타일의 기능을 분담시킨것이라고 생각할 수 있겠다.
